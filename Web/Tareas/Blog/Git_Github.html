¿Cómo funcionan Git y Github?

Git es un sistema de control de versiones de código abierto. Fue diseñado para gestionar y los cambios que se hagan en los archivos. tiene soporte para trabajar en diferentes ramas, lo que lo hace ideal para el manejo de proyectos largos.

Git Hub por otra parte es una plataforma que permite guardar código. Cualidad que a desarroladores poder colaborar en distintos proyectos. Este permite poder tener version de las versiones permitiendo poder rastrear errores, poder agregar caracterizticas al software, para cada proyecto.

El flujo de trabajo de estos empieza con la creación de un repositorio, ya sea en Github lo que crea un repositorio remoto, o en Git creandolo de manera local en tu computadora. En caso de tener un repositorio remoto puedes clonarlo en tu computadora y hacer cambios de manera local en ese repositorio. Para los cambios que se hagan de manera local en el repositorio deberán ser puestos en el área de staging, en la cual se mostraran todos los cambios que se subiran al repositorio.

Estos cambios se haran en la rama en la que se este trabajando. Esto aplica para trabajos locales y directamente en el repositorio remoto

Posterior a esto se dejará evidencia del cambio realizado, donde es recomendable dejar un comentario acerca de los cambios realizados. Después de eso, en caso de trabar remotamente con un repositorio clonado, puedes subir los cambios en la nube.

Comandos usados en Git

Los siguientes comandos son usados en la terminal de Linux y Mac, y en caso de windows es necesario el instalar Git Bash

Link de la página de Git

Comandos Descripción
git init Este comando crea un repositorio de Git en el repositorio en el directorio en el que estas
git clone Este comando es usado para clonar un repositorio remoto en tu computadora
git status Se usa para ver que cambios estan en el área para ser subidos de un repositorio remoto
git add Añade los cambios realizados en el repositorio local a la cola para subirlos al repositorio remoto
git commit .m "comentario" Sube los cambios que estaban en el área de cambios listos, en conjunto con un comentario que describa los cambios realizados
git log Te muestra el historial de todos los commits que se han hecho en el repositorio, tiene los datos del autor y los comentarios hechos
git pull Este comando compara los cambios del repositorio remoto con los del local y los combina en tu repositorio local
git push -u origin {nombre de la rama} Sube los cambios que estan hechos de manera local al repositorio remoto
git branch {nombre de la rama} Crea una rama nueva de trabajo con la versión actual del repositorio local
git branch -a Muestra todas las ramas que se crearon en el repositorio
git checkout -b {nombre de la rama} Crea una rama nueva de trabajo con la versión actual del repositorio local y te cambia a ella automaticamente
git checkout {nombre de la rama} Te cambia a la rama que indiques en el nombre
git merge {nombre de la rama} Compara y combina los contenidos de la rama en la que estás y la rama que indiques en el nombre
git branch -d {nombre de la rama} Borra la rama del repoitorio local que indiques en el nombre
git pus origin -d {nombre de la rama} Borra la rama del repositorio remoto
git restore Sirve para restaurar un commit que se hizo por error
git remove {nombre del archivo/directorio} Quita de la cola para subir los cambios los cambios indicados
